Artifacts is a central term in Serverless cloud. There are different types of artifacts, such as code artifacts, database artifacts and storage artifacts. Those are used to deploy new versions and created when requesting backups of database and storage.

## Create artifacs

There are different types of artifacts, such as code artifacts, database artifacts and storage artifacts. Read more about how to:

- [Create .NET artifact](/cloud/serverless/artifacts-create-dotnet-artifact)
- [Create Next.js artifact](/cloud/serverless/artifacts-create-nextjs-artifact)
- [Create Nuxt.js artifact](/cloud/serverless/artifacts-create-nuxtjs-artifact)
- [Create Node.js artifact](/cloud/serverless/artifacts-create-nodejs-artifact)
- [Create Database artifacts](/cloud/serverless/artifacts-create-sql-backup-artifact)
- [Create Storage artifacts](/cloud/serverless/artifacts-create-storage-backup-artifact)

## [Deploy](/cloud/serverless/artifacts-deploy-artifact)

Deploy command is used to deploy a code artifact to an app. Note that database artifact and storage artifacts is only possible to apply when installing an app for the first time. [Read more about how to deploy an artifact](/cloud/serverless/artifacts-deploy-artifact).

## [Download](/cloud/serverless/artifacts-download-artifact)

You can download artifacts to your computer, you find artifacts by subscription or show specific app to get which current artifact id that are being used.
