# What's new in 8.10

The Storefront API as second release candidate with ability to query for product and category relations. Improved user features for order management. The React accelerator beta 3 is released at the same time with new abilities such as quick search, search result page and category page.

## Highlights

- React accelerator (beta 3)
- Quick search (React accelerator)
- Search result page (React accelerator)
- Category page (React accelerator)
- Litium Storefront API (release candidate 2)
- Ability to filter orders on channels
- Ability to filter orders on tags
- Ablilty to see additional order information in the order view
- Ablilty to see tags for orders, payments and deliveries in the order view
- Ability to set permissions also to local sub blocks
- Ability to query Storefront API for product and category relations

## React accelerator (third beta)

The new react accelerator has its own description and release notes, but here are some of the new things delivered this time.

The React accelerator is designed mobile first for a better and updated user experience.

## Search result page (React accelerator)

Horizontal tabs are introduced for a quick navigation between hits for products, categories and pages. An updated UI for the sorting panel that takes over the screen and slides up from bottom gives a clear and modern experience.

The filter section is in progress and will also get an updated UI in a coming release.

## Category page (React accelerator)

The design of the category page is in line with the design scheme of the search result page to make product listing consistent to the user. The button for filter and sorting and the numbers indicating the product hits are placed in the same way in both search views as in the category page.

A new navigation for child categories is introduced.

## Improved UI for order management

Several improvements are done for managing orders.

### Ability to filter orders on channels and on order tags

Users can now filter orders on channels and order tags to easier find what hen is looking for.

![Skärmavbild 2023-05-29 kl. 14.16.27.png](/images/platform/whats-new/Ska%CC%88rmavbild%202023-05-29%20kl.%2014.16.27.png)

### Ablilty to see additional order information in the order view

Additional order information is now visible in the order view. To make the order view clean from information unnecessary to users, there is a setting for administrators to define which additional order info keys that should be visible to users in back office.

![Skärmavbild 2023-05-29 kl. 14.21.30.png](/images/platform/whats-new/Ska%CC%88rmavbild%202023-05-29%20kl.%2014.21.30.png)

### Ablilty to see tags for orders, payments and deliveries in the order view

Users will now see any tags for orders, payments or deliveries on the order view.
