- ✅ [Installed](/getting-started/installation) the MCP server

- ✅ [Set up](/getting-started/authentication) OAuth2 credentials

## Your First Tool Call

Let's make your first tool call to list all websites in your Litium instance.

<Tabs>
  <Tab title="Using Cursor">
    <Steps>
      <Step title="Open Cursor">
        Launch Cursor and open any project or create a new one
      </Step>

      <Step title="Start AI Chat">
        Open the AI chat panel (Cmd/Ctrl+L)
      </Step>

      <Step title="Ask for Websites">
        Type: "Can you list all websites in my Litium instance?"
      </Step>

      <Step title="Review Response">
        Cursor will use the `manage_website` tool and show you the results:
        
        ```json
        {
          "websites": [
            {
              "systemId": "abc-123",
              "name": "Main Store",
              "url": "https://store.example.com"
            }
          ]
        }
        ```
      </Step>
    </Steps>
  </Tab>

  <Tab title="Using Claude Desktop">
    <Steps>
      <Step title="Open Claude Desktop">
        Launch Claude Desktop application
      </Step>

      <Step title="Start Conversation">
        Start a new conversation
      </Step>

      <Step title="Ask for Websites">
        Type: "Can you list all websites in my Litium instance?"
      </Step>

      <Step title="Review Response">
        Claude will use the `manage_website` tool and format the response for you
      </Step>
    </Steps>
  </Tab>

  <Tab title="Using cURL">
    <Steps>
      <Step title="List Available Tools">
        First, verify your connection:
        
        ```bash
        curl -X POST http://localhost:3000/api/mcp \
          -H "Content-Type: application/json" \
          -H "Accept: application/json" \
          -H "X-Litium-Base-Url: https://demoadmin.litium.com" \
          -H "X-Litium-Client-Id: your-client-id" \
          -H "X-Litium-Client-Secret: your-client-secret" \
          -d '{
            "jsonrpc": "2.0",
            "method": "tools/list",
            "id": 1
          }'
        ```
      </Step>

      <Step title="Call Manage Website Tool">
        Make your first tool call:
        
        ```bash
        curl -X POST http://localhost:3000/api/mcp \
          -H "Content-Type: application/json" \
          -H "Accept: application/json" \
          -H "X-Litium-Base-Url: https://demoadmin.litium.com" \
          -H "X-Litium-Client-Id: your-client-id" \
          -H "X-Litium-Client-Secret: your-client-secret" \
          -d '{
            "jsonrpc": "2.0",
            "method": "tools/call",
            "params": {
              "name": "manage_website",
              "arguments": {
                "operation": "list"
              }
            },
            "id": 2
          }'
        ```
      </Step>

      <Step title="Review Response">
        You'll receive a JSON-RPC response with your websites
      </Step>
    </Steps>
  </Tab>
</Tabs>

## More Examples

### Search for Products

<CodeGroup>
```bash cURL
curl -X POST http://localhost:3000/api/mcp \
  -H "Content-Type: application/json" \
  -H "X-Litium-Base-Url: https://demoadmin.litium.com" \
  -H "X-Litium-Client-Id: your-client-id" \
  -H "X-Litium-Client-Secret: your-client-secret" \
  -d '{
    "jsonrpc": "2.0",
    "method": "tools/call",
    "params": {
      "name": "manage_product",
      "arguments": {
        "operation": "search",
        "params": {
          "search": "dress",
          "take": 10
        }
      }
    },
    "id": 3
  }'
```

```text Natural Language (Cursor/Claude)
"Can you search for dress products in my catalog?"
```
</CodeGroup>

**Tested**: Returns products like "Court Dress", "W Woven Dress" ✅

## Common Workflows

<AccordionGroup>
  <Accordion title="Product Management Workflow" icon="box">
    1. **Search for products**: `manage_product` with operation `search`
    2. **Get product details**: `manage_product` with operation `get`
    3. **Update product**: `manage_product` with operation `update`
    4. **Verify update**: `manage_product` with operation `get` again
  </Accordion>

  <Accordion title="Content Management Workflow" icon="file-lines">
    1. **List blocks**: `manage_block` with operation `search`
    2. **Get specific block**: `manage_block` with operation `get`
    3. **Update content**: `manage_block` with operation `update`
    4. **Publish changes**: Verify with `manage_block` operation `get`
  </Accordion>

  <Accordion title="Customer Support Workflow" icon="headset">
    1. **Find order**: `manage_order` with operation `search` (e.g., "LS10070")
    2. **Get order details**: `manage_order` with operation `get`
    3. **Find customer**: `manage_customer` with operation `search`
    4. **Get customer details**: `manage_customer` with operation `get`
    
    **Tested Example**: Search for "LS10070" → Returns exact order ✅
  </Accordion>
</AccordionGroup>

## Understanding Tool Responses

All tools return data in JSON format. Here's what a typical response looks like:

```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "result": {
    "content": [
      {
        "type": "text",
        "text": "{\"websites\":[{\"systemId\":\"abc-123\",\"name\":\"Main Store\"}]}"
      }
    ]
  }
}
```

<Info>
  When using AI assistants like Cursor or Claude, they automatically parse and format these responses for you in a readable way.
</Info>
