## Usage Patterns

All tools follow a consistent pattern with an `operation` parameter:

### Example: Manage Product

```json
// Search products
{
  "tool": "manage_product",
  "arguments": {
    "operation": "search",
    "params": {
      "search": "laptop",
      "take": 10
    }
  }
}

// Get specific product
{
  "tool": "manage_product",
  "arguments": {
    "operation": "get",
    "systemId": "abc-123-def"
  }
}

// Create new product
{
  "tool": "manage_product",
  "arguments": {
    "operation": "create",
    "data": {
      "name": "New Product",
      "price": 99.99
    }
  }
}
```

### Common Operations

Most tools support these standard operations:

| Operation | Description | Required Parameters |
|-----------|-------------|---------------------|
| `search` | Search with filters | `params` (optional) |
| `get` | Get by ID | `systemId` |
| `create` | Create new | `data` |
| `update` | Update existing | `systemId`, `data` |
| `delete` | Delete by ID | `systemId` |

### Search Parameters

Search operations typically support:

```json
{
  "search": "keyword",   // Search term (optional)
  "skip": 0,            // Pagination offset (default: 0)
  "take": 20,           // Results per page (default: 20)
  "sort": "name"        // Sort field (optional)
}
```

All tools return responses in MCP format:

```json
{
  "content": [
    {
      "type": "text",
      "text": "{...}"    // JSON data
    }
  ]
}
```

### Read-Only vs. Modify Operations

<Tabs>
  <Tab title="Read-Only Operations (Safe)">
    These operations only retrieve data:
    - `search` - Find resources
    - `get` - Get by ID
    - `list` - List all
    - `get_*` variants (e.g., `get_variants`, `get_subcategories`)
  </Tab>

  <Tab title="Modify Operations (Use with Care)">
    These operations change data in Litium:
    - `create` - Add new resources
    - `update` - Modify existing
    - `delete` - Remove resources
    - `unpublish` - Unpublish pages
    - `publish_draft` - Publish drafts
    
    <Warning>
      Always verify IDs before using modify operations in production!
    </Warning>
  </Tab>
</Tabs>

## Benefits of Domain-Based Tools

The consolidated tool structure provides:

1. **Fewer Tools** - 18 instead of 100+ individual tools
2. **Consistent Interface** - All tools use the `operation` parameter
3. **Better Organization** - Grouped by business domain
4. **Easier Discovery** - Find related operations in one tool
5. **Reduced Complexity** - Simpler for AI models to use
