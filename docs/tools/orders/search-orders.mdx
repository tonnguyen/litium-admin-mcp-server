## Overview

Search for sales orders in your Litium instance using flexible search criteria.

<Info>
  This is a **read-only** tool that does not modify any data.
</Info>

## Parameters

<ParamField path="search" type="string" optional>Search term for filtering orders by order number, customer name, or email</ParamField>
<ParamField path="skip" type="number" optional default="0">Pagination offset</ParamField>
<ParamField path="take" type="number" optional default="20">Results per page</ParamField>
<ParamField path="sort" type="string" optional>Sort field</ParamField>

## Response

Returns a paginated list of orders matching the search criteria.

<ResponseField name="results" type="array">
  Array of order objects
  
  <Expandable title="Order Object">
    <ResponseField name="systemId" type="string">Unique order identifier</ResponseField>
    <ResponseField name="orderNumber" type="string">Order number</ResponseField>
    <ResponseField name="customerName" type="string">Customer name</ResponseField>
    <ResponseField name="totalAmount" type="number">Order total</ResponseField>
    <ResponseField name="status" type="string">Order status</ResponseField>
    <ResponseField name="orderDate" type="string">Order date</ResponseField>
  </Expandable>
</ResponseField>

## Troubleshooting

<AccordionGroup>
  <Accordion title="No results found">
    **Cause**: Search term doesn't match any orders
    
    **Solution**: 
    - Verify the order number is correct
    - Try searching by customer email or name
    - Remove search term to list all recent orders
  </Accordion>

  <Accordion title="Too many results">
    **Cause**: Search is too broad
    
    **Solution**: Use more specific search criteria or reduce `take` parameter
  </Accordion>

  <Accordion title="Authentication error">
    **Cause**: Invalid OAuth2 credentials
    
    **Solution**: Verify credentials in MCP client configuration
  </Accordion>
</AccordionGroup>
