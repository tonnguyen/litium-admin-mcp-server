## Overview

Retrieve detailed information about a specific sales order using its system ID.

<Info>
  This is a **read-only** tool that does not modify any data.
</Info>

## Parameters

<ParamField path="systemId" type="string" required>
  The unique system ID of the order to retrieve
</ParamField>

## Response

Returns complete order details including items, customer, shipping, and payment information.

<ResponseField name="systemId" type="string">Order identifier</ResponseField>
<ResponseField name="orderNumber" type="string">Order number</ResponseField>
<ResponseField name="orderLines" type="array">Order items/products</ResponseField>
<ResponseField name="customer" type="object">Customer information</ResponseField>
<ResponseField name="shippingInfo" type="object">Shipping details</ResponseField>
<ResponseField name="paymentInfo" type="object">Payment details</ResponseField>
<ResponseField name="totalAmount" type="number">Order total</ResponseField>
<ResponseField name="status" type="string">Order status</ResponseField>

## Order Information Breakdown

The order response includes several key sections:

### Order Lines
Details about each item in the order:
- Product ID and name
- Quantity ordered
- Unit price and total

### Customer Information
Customer details:
- Full name
- Email address
- Phone number (if available)

### Shipping Information
Delivery details:
- Full shipping address
- Shipping method
- Estimated delivery date

### Payment Information
Payment details:
- Payment method used
- Payment status
- Transaction ID (if available)

## Troubleshooting

<AccordionGroup>
  <Accordion title="Order not found (404)">
    **Cause**: The system ID doesn't exist or is incorrect
    
    **Solution**: 
    - Verify the systemId using [search_orders](/tools/orders/search-orders)
    - Ensure no typos in the ID
    - Check that the order exists in your Litium instance
  </Accordion>

  <Accordion title="Access denied (403)">
    **Cause**: Insufficient permissions to view orders
    
    **Solution**: Verify your OAuth2 credentials have read permissions for orders
  </Accordion>

  <Accordion title="Invalid system ID format">
    **Cause**: System ID is not in the correct format
    
    **Solution**: Use the full system ID as returned by search_orders, not the order number
  </Accordion>
</AccordionGroup>
