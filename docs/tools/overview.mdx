## Common Patterns

All tools follow consistent patterns for easy use:

### Search Tools

Search tools accept optional parameters for filtering and pagination:

```json
{
  "search": "keyword",     // Search term (optional)
  "skip": 0,              // Pagination offset (default: 0)
  "take": 20,             // Results per page (default: 20)
  "sort": "name"          // Sort field (optional)
}
```

### Get Tools

Get tools retrieve a single resource by its system ID:

```json
{
  "systemId": "abc-123-def-456"  // Unique identifier
}
```

### Create Tools

Create tools accept a resource object:

```json
{
  "product": {              // Resource type varies
    "name": "Product Name",
    "price": 99.99,
    // ... other fields
  }
}
```

### Update Tools

Update tools require both the system ID and updated data:

```json
{
  "systemId": "abc-123-def-456",
  "product": {
    "price": 89.99        // Updated fields
  }
}
```

### Delete Tools

Delete tools only need the system ID:

```json
{
  "systemId": "abc-123-def-456"
}
```

## Response Format

All tools return responses in JSON-RPC format:

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "content": [
      {
        "type": "text",
        "text": "{...}"    // JSON string with actual data
      }
    ]
  }
}
```

<Info>
  When using AI assistants like Cursor or Claude, they automatically parse and format these responses for you.
</Info>

### Read-Only vs. Modify Tools

<Tabs>
  <Tab title="Read-Only Tools (Safe)">
    These tools only retrieve data and don't modify anything:
    - All `search_*` tools
    - All `get_*` tools
    - All API discovery/documentation tools
  </Tab>

  <Tab title="Modify Tools (Use with Care)">
    These tools change data in Litium:
    - All `create_*` tools
    - All `update_*` tools
    - All `delete_*` tools
    
    <Warning>
      Always verify IDs before using modify tools in production!
    </Warning>
  </Tab>
