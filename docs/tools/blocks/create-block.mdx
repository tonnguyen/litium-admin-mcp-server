## Overview

Create a new content block in your Litium instance. This tool allows you to add new content blocks that can be used across your websites.

<Warning>
  This tool **modifies data** in Litium. Always verify the block data before creating.
</Warning>

## Parameters

<ParamField path="block" type="object" required>
  Complete block data object to create
  
  <Expandable title="Block Object Properties">
    <ParamField path="name" type="string" required>
      Display name for the block
    </ParamField>
    
    <ParamField path="fieldTemplateSystemId" type="string" required>
      System ID of the field template to use for this block
    </ParamField>
    
    <ParamField path="content" type="object" optional>
      Block content data (HTML, text, etc.)
    </ParamField>
    
    <ParamField path="fields" type="object" optional>
      Custom field values for the block
    </ParamField>
  </Expandable>
</ParamField>

## Response

Returns the newly created block with its system ID.

<ResponseField name="systemId" type="string">
  Unique identifier assigned to the new block
</ResponseField>

<ResponseField name="name" type="string">
  Display name of the created block
</ResponseField>

<ResponseField name="created" type="string">
  Creation timestamp (ISO 8601 format)
</ResponseField>

<ResponseField name="createdBy" type="string">
  Username of the creator
</ResponseField>

## Field Template IDs

<Note>
  You'll need to obtain valid field template system IDs from your Litium instance. These templates define the structure and available fields for blocks.
</Note>

Common field template types:
- **Banner Template**: For hero banners and promotional content
- **Text Template**: For text-based content blocks
- **Image Template**: For image galleries and media blocks
- **Custom Templates**: Your organization's custom templates

## Troubleshooting

<AccordionGroup>
  <Accordion title="Invalid field template">
    **Cause**: The fieldTemplateSystemId doesn't exist or is incorrect
    
    **Solution**: Verify the template ID exists in your Litium instance. Check with your Litium administrator for valid template IDs.
  </Accordion>

  <Accordion title="Required field missing">
    **Cause**: The template requires fields that weren't provided
    
    **Solution**: Check the field template requirements and ensure all required fields are included in the `fields` object.
  </Accordion>

  <Accordion title="Validation error">
    **Cause**: Block data doesn't meet validation requirements
    
    **Solution**: Ensure the `name` field is unique and all content follows the template's validation rules.
  </Accordion>

  <Accordion title="Permission denied">
    **Cause**: Insufficient permissions to create blocks
    
    **Solution**: Verify your OAuth2 credentials have write permissions for blocks.
  </Accordion>
</AccordionGroup>
