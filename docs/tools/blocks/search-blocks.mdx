## Overview

Search for content blocks in your Litium instance using flexible search and pagination parameters. This tool uses the Litium Admin API endpoint `GET /Litium/api/admin/blocks/blocks/search`.

<Info>
  This is a **read-only** tool that does not modify any data.
</Info>

## Parameters

<ParamField path="search" type="string" optional>
  Search term for filtering blocks by name, content, or ID
  
  **Example**: `"header"`, `"footer"`, `"banner"`
</ParamField>

<ParamField path="skip" type="number" optional default="0">
  Number of items to skip for pagination
  
  **Example**: `0`, `20`, `40`
</ParamField>

<ParamField path="take" type="number" optional default="20">
  Number of items to return per page (max: 100)
  
  **Example**: `10`, `20`, `50`
</ParamField>

<ParamField path="sort" type="string" optional>
  Field to sort results by
  
  **Example**: `"name"`, `"created"`, `"modified"`
</ParamField>

## Response

Returns a paginated list of content blocks matching the search criteria.

<ResponseField name="results" type="array">
  Array of content block objects
  
  <Expandable title="Block Object">
    <ResponseField name="systemId" type="string">
      Unique identifier for the block
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Display name of the block
    </ResponseField>
    
    <ResponseField name="content" type="object">
      Block content data
    </ResponseField>
    
    <ResponseField name="created" type="string">
      Creation timestamp (ISO 8601)
    </ResponseField>
    
    <ResponseField name="modified" type="string">
      Last modification timestamp (ISO 8601)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="total" type="number">
  Total number of blocks matching the search
</ResponseField>

## Troubleshooting

<AccordionGroup>
  <Accordion title="No results returned">
    **Cause**: Search term doesn't match any blocks
    
    **Solution**: Try a broader search term or remove the search parameter to list all blocks
  </Accordion>

  <Accordion title="Too many results">
    **Cause**: Search is too broad
    
    **Solution**: Use a more specific search term or use pagination with `take` parameter
  </Accordion>

  <Accordion title="Authentication error">
    **Cause**: Invalid or missing OAuth2 credentials
    
    **Solution**: Verify your credentials in the MCP client configuration
  </Accordion>
</AccordionGroup>
